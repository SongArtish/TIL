## 학습 목표

1. 사용자를 관찰하고 이해하는 방법
2. 인사이트가 해결책으로 구체화되는 과정
3. 사용자의 행동을 유도하는 방법
4. 데이터 기반 의사결정



## What is UX?

> UX는 1) **사용자를 이해**하고, 2) **사용자의 결핍을 발견**하고,3) **문제를 해결**하는데 도움을 주는 도구이다.



## UX/UI 디자이너의 역할

### 1) 전통적인 역할

- 제품의 시각적인 부분을 아름답게 표현

### 2) 새롭게 요구되는 역할

- 프로젝트 전반에서 종합적으로 디자인에 참여

```markdown
- 사용자 리서치
- UX 기획/설계 = 스토리보드 작성
- 와이어프레임
- UI 디자인
- 프로토타이핑
- 프로젝트 진행(커뮤니케이션)
- 브랜딩 및 편집 디자인
- HTML/CSS 코딩
```
